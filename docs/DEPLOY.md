# Как запустить дашборд продаж Kaspi — инструкция для новичка

Не пугайся длины инструкции — каждый шаг простой, просто их много. Делай по порядку, ничего не пропускай. Если что-то не сработает — просто скопируй текст ошибки и пришли мне, я помогу разобраться.

Всего тебе нужно зарегистрироваться в **3 бесплатных сервисах**:
1. **Neon** — здесь будет жить база данных (хранилище твоих заказов)
2. **Render** — здесь будет жить backend (программа, которая ходит в Kaspi и считает статистику)
3. **Vercel** — здесь будет жить сам сайт, который ты открываешь в браузере

И тебе понадобится аккаунт на **GitHub** — это место, куда загружается код, чтобы Render и Vercel могли его оттуда забрать.

---

## Шаг 0. Получи токен Kaspi API

1. Зайди в кабинет продавца на kaspi.kz
2. Перейди в **Настройки → Токен API**
3. Нажми **«Сформировать»**
4. Скопируй токен куда-нибудь в заметки — он понадобится на шаге 4. Никому его не показывай и не отправляй — это как пароль от магазина.

---

## Шаг 1. Создай аккаунт на GitHub и загрузи туда код

1. Зайди на https://github.com и зарегистрируйся (если ещё нет аккаунта)
2. Нажми **New repository** (зелёная кнопка)
3. Назови репозиторий, например `kaspi-dashboard`, выбери **Private** (приватный — чтобы код не видели посторонние), нажми **Create repository**
4. Тебе нужно загрузить туда папки `backend` и `frontend`, которые я подготовил. Самый простой способ без консоли:
   - На странице репозитория нажми **uploading an existing file**
   - Перетащи туда все файлы из папки `backend` (включая скрытый файл — но **.env.example можно**, а сам `.env` с твоим токеном никогда никуда не загружай)
   - Сделай то же самое для `frontend` — но проще: сначала создай в репозитории папки `backend` и `frontend` (просто впиши `backend/имя_файла` при загрузке, GitHub сам создаст папку)

Если этот способ покажется неудобным — скажи мне, и я подскажу альтернативный путь через специальную программу GitHub Desktop с понятным интерфейсом (без консоли).

---

## Шаг 2. Создай базу данных на Neon

1. Зайди на https://neon.tech и зарегистрируйся (можно через GitHub-аккаунт — это свяжет всё в один клик)
2. Создай новый проект (**Create a project**), любое название, регион — любой ближайший (например, Europe)
3. После создания Neon покажет тебе строку **Connection string** — она выглядит так:
   `postgresql://имя:пароль@адрес.neon.tech/имя_базы?sslmode=require`
4. **Скопируй эту строку** — это и есть значение для `DATABASE_URL`, которое понадобится на следующем шаге

---

## Шаг 3. Разверни backend на Render

1. Зайди на https://render.com и зарегистрируйся через GitHub-аккаунт
2. Нажми **New → Web Service**
3. Выбери репозиторий `kaspi-dashboard`, который ты создал на шаге 1
4. В настройках укажи:
   - **Root Directory**: `backend`
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: Free
5. Дальше нужно добавить переменные окружения (это и есть содержимое файла `.env`) — найди раздел **Environment Variables** и добавь по одной:
   - `KASPI_API_TOKEN` → вставь токен из Шага 0
   - `DATABASE_URL` → вставь строку из Шага 2
   - `DASHBOARD_PASSWORD` → придумай свой пароль для входа на сайт
6. Нажми **Create Web Service**. Render начнёт собирать и запускать backend — это займёт пару минут
7. Когда всё готово, наверху страницы появится адрес вида `https://kaspi-dashboard-backend.onrender.com` — **скопируй его**, он понадобится на следующем шаге

> Важный нюанс бесплатного тарифа Render: если сайтом не пользоваться 15 минут, сервер "засыпает" и следующий запрос будет грузиться на 30-50 секунд дольше обычного, пока он "проснётся". Для ежедневного личного использования это не страшно — просто первое открытие сайта в день может быть чуть медленнее.

---

## Шаг 4. Разверни фронтенд (сам сайт) на Vercel

1. Зайди на https://vercel.com и зарегистрируйся через GitHub-аккаунт
2. Нажми **Add New → Project**
3. Выбери репозиторий `kaspi-dashboard`
4. В настройках укажи:
   - **Root Directory**: `frontend`
   - Framework Preset: Vercel определит **Vite** автоматически
5. В разделе **Environment Variables** добавь:
   - `VITE_API_URL` → вставь адрес backend из Шага 3 (например `https://kaspi-dashboard-backend.onrender.com`)
6. Нажми **Deploy**. Через минуту-две Vercel выдаст тебе адрес сайта вида `https://kaspi-dashboard.vercel.app`

**Это и есть твой дашборд!** Открывай эту ссылку каждый день — можно добавить на главный экран телефона, чтобы было как приложение.

---

## Шаг 5. Первый вход и проверка

1. Открой ссылку с Vercel
2. Введи пароль, который ты задал в Шаге 3 (`DASHBOARD_PASSWORD`)
3. Если всё подключено правильно — ты увидишь дашборд. Если данных пока нет (пустые графики) — нажми кнопку **«Обновить сейчас»**, чтобы запустить первую синхронизацию с Kaspi вручную

Дальше синхронизация будет происходить сама каждую ночь — заходить и нажимать ничего не нужно.

---

## Если что-то не работает

- **Сайт открывается, но после ввода пароля ошибка** — скорее всего, неправильно указан `VITE_API_URL` на Vercel, или backend на Render ещё не проснулся (подожди 30 секунд и обнови страницу)
- **Графики пустые** — нажми «Обновить сейчас»; если всё равно пусто, скорее всего нет заказов за выбранный период, или токен Kaspi указан неверно
- **Любая другая ошибка** — открой ссылку на Render, раздел **Logs**, скопируй последние строки с ошибкой и пришли мне

Я могу пройти этот процесс с тобой шаг за шагом, если что-то будет неясно — просто пиши, на каком шаге застряла.
